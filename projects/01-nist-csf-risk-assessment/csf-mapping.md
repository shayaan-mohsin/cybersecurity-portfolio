# NIST CSF Mapping

| NIST CSF function | Portfolio interpretation | Example action |
|---|---|---|
| Govern | Define security accountability, policy ownership, and risk decisions | Assign owners for access reviews, vendor access, and incident response |
| Identify | Understand assets, workflows, users, vendors, and sensitive data | Build a lightweight inventory of SaaS tools, endpoints, and business owners |
| Protect | Reduce risk through access control, training, and safeguards | Apply least privilege, MFA, and security awareness practices |
| Detect | Improve visibility into suspicious activity and reporting | Define alert review and phishing reporting process |
| Respond | Prepare for incidents through roles, escalation, and communication | Create an incident response playbook |
| Recover | Restore operations and learn from disruption | Define recovery objectives and test restoration steps |

## Key Observation

The strongest near-term value comes from governance and process improvements. These actions are realistic for a smaller organization and create a foundation for more technical controls later.
