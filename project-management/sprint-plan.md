# Sprint Plan

## Cadence

- Sprint length: one week
- Review rhythm: end of each sprint
- Planning rhythm: start of each sprint
- Portfolio focus: quality over volume

## Definition Of Ready

A task is ready when:

- The deliverable is clear
- The source or scenario is identified
- The output format is known
- No sensitive or private material is required

## Definition Of Done

A deliverable is done when:

- It is written in a public-safe format
- It cites or names authoritative frameworks/sources when relevant
- It avoids overclaiming hands-on enterprise experience
- It is linked from the relevant project README
- It can be explained in an interview

## Sprint Review Template

Use this at the end of each sprint:

- What was completed?
- What evidence was published?
- What changed in the backlog?
- What risk or blocker remains?
- What should be prioritized next?
