# Portfolio Roadmap

## Sprint 0: Foundation

Goal: establish the repository, portfolio structure, GitHub Project workflow, and public-facing positioning.

Deliverables:
- Repository README
- Project folders
- GitHub Project field plan
- Backlog seed list
- Issue templates

## Sprint 1: Healthcare Breach Risk Assessment

Goal: publish a GRC/risk assessment using public HHS OCR breach records, HHS healthcare cybersecurity guidance, and NIST CSF 2.0 framing.

Deliverables:
- Data methodology
- Breach trend analysis
- Evidence-informed risk register
- NIST CSF mapping
- Prioritized 30/60/90-day roadmap
- Executive brief

## Sprint 2: CISA KEV Vulnerability Prioritization

Goal: publish an exploited-vulnerability prioritization workflow using the official CISA KEV catalog.

Deliverables:
- Data source notes
- KEV catalog analysis
- Triage scoring model
- Remediation SLA matrix
- Python analysis workflow
- Weekly vulnerability brief
- Executive summary

## Sprint 3: MITRE ATT&CK CTI Brief

Goal: demonstrate CTI-style research, adversary behavior mapping, and response recommendations.

Deliverables:
- Threat profile
- ATT&CK mapping
- Business impact summary
- Detection and response recommendations
- Incident-response improvement backlog

## Sprint 4: Portfolio Polish

Goal: refine the repository for public review and ensure each deliverable is accurate, evidence-backed, and professionally bounded.

Deliverables:
- Final README polish
- Evidence review
- Executive summaries
- Public content review
- LinkedIn/GitHub profile alignment
