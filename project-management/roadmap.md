# Portfolio Roadmap

## Sprint 0: Foundation

Goal: establish the repository, portfolio structure, GitHub Project workflow, and public-facing positioning.

Deliverables:
- Repository README
- Project folders
- GitHub Project field plan
- Backlog seed list
- Issue templates

## Sprint 1: NIST CSF Risk Assessment

Goal: publish the first GRC/risk case study using a fictional organization and NIST CSF 2.0 framing.

Deliverables:
- Scope and assumptions
- Asset and risk register
- NIST CSF mapping
- Prioritized risk roadmap
- Executive brief

## Sprint 2: CISA KEV Vulnerability Prioritization

Goal: demonstrate vulnerability prioritization using authoritative public vulnerability intelligence.

Deliverables:
- Data source notes
- Triage model
- Remediation SLA matrix
- Python/pandas analysis workflow
- Weekly vulnerability brief

## Sprint 3: MITRE ATT&CK CTI Brief

Goal: demonstrate CTI-style research, adversary behavior mapping, and response recommendations.

Deliverables:
- Threat profile
- ATT&CK mapping
- Business impact summary
- Detection and response recommendations
- Incident-response improvement backlog

## Sprint 4: Portfolio Polish

Goal: turn the work into a recruiter-ready cybersecurity portfolio.

Deliverables:
- Final README polish
- Evidence review
- Executive summaries
- Public-safe content review
- LinkedIn/GitHub profile alignment
