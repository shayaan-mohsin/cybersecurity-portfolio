# Portfolio Risk Register

| Risk | Likelihood | Impact | Response |
|---|---|---|---|
| Portfolio sounds too academic | Medium | High | Include executive summaries, PM artifacts, and practical recommendations |
| Projects overclaim enterprise experience | Medium | High | Label scenarios as fictional/sanitized and describe work as portfolio analysis |
| Scope gets too broad | High | Medium | Keep each project to one clear use case and one main deliverable set |
| Public content exposes course-restricted material | Low | High | Rewrite all work as sanitized summaries, not copied assignments |
| Technical artifacts are thin | Medium | Medium | Add small Python/pandas workflow and structured analysis tables |
| Project management evidence is hidden | Medium | Medium | Maintain GitHub Project views, backlog, sprint reviews, and roadmap artifacts |
