# GitHub Project Setup

Create a GitHub Project named `Cybersecurity Portfolio Roadmap`.

## Project Fields

| Field | Type | Values |
|---|---|---|
| Status | Single select | Backlog, Ready, In Progress, Review, Done |
| Project Area | Single select | GRC, Vulnerability Risk, CTI, PM |
| Deliverable Type | Single select | Research, Analysis, Report, Dashboard, README, Review |
| Priority | Single select | High, Medium, Low |
| Sprint | Iteration | One-week iterations |
| Evidence | Single select | Needs Evidence, Drafted, Reviewed, Published |

## Recommended Views

| View | Purpose |
|---|---|
| Backlog | All planned work grouped by Project Area |
| Current Sprint | Current sprint items grouped by Status |
| Roadmap | Sprint-level sequencing across the portfolio |
| Portfolio Evidence | Deliverables filtered to Reviewed or Published |
| PM Dashboard | Work grouped by Priority and Project Area |

## Suggested Labels

- `area:grc`

Use [backlog.csv](backlog.csv) as the starter issue list. Each row can become one GitHub issue and one GitHub Project item.
